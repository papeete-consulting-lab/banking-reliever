> **Mode B (contract + stub), Python flavour.**

> **⚠ Partial supersession (TASK-002 merged).** The **MINT** path
> (`POST /anchors`) and the **GET** path (`GET /anchors/{internal_id}`) are
> now served by the real microservice under
> [`../backend/`](../backend/README.md). This stub remains useful for the
> **UPDATED / ARCHIVED / RESTORED / PSEUDONYMISED** transitions until
> TASK-003 … TASK-005 land. Downstream consumers should target the real
> service for MINT + GET and continue using this stub for the lifecycle
> verbs they need before the corresponding task ships.

This is a **development stub** — it materialises the consumer-facing surface
of `BNK.RLVR.CAP.SUP.002.BEN` (the canonical Beneficiary Identity Anchor) so that
downstream consumer capabilities (SCO, ENR, ENV, DSH, VIE, B2B.FLW, AUD, RET,
DAT) can develop against the real wire format and HTTP shape **before** the
real microservice ships under TASK-002+.

It is **not** a production service. It has no persistence, no JWT verification,
no domain logic. It serves canned cold data and emits synthetic events.

## What it does

### Publisher half — RabbitMQ resource events
- Declares the topic exchange `sup.002.ben-events` (durable, owned by
  `BNK.RLVR.CAP.SUP.002.BEN`) per `ADR-TECH-STRAT-001` Rule 1, 5.
- Emits `BNK.RLVR.RVT.SUP.002.BENEFICIARY_ANCHOR_UPDATED` on routing key
  `BNK.RLVR.EVT.SUP.002.BENEFICIARY_ANCHOR_UPDATED.BNK.RLVR.RVT.SUP.002.BENEFICIARY_ANCHOR_UPDATED`
  (ADR-TECH-STRAT-001 Rule 4).
- Cycles deterministically through the 5 `transition_kind` values declared in
  `process/BNK.RLVR.CAP.SUP.002.BEN/bus.yaml`: `MINTED`, `UPDATED`, `ARCHIVED`,
  `RESTORED`, `PSEUDONYMISED`. The `PSEUDONYMISED` payloads exercise the
  conditional `if/then` of the JSON Schema (PII fields null, `right_exercise_id`
  + `pseudonymized_at` required).
- Every envelope carries a fresh UUIDv7 trio (`message_id`, `correlation_id`,
  `causation_id`) per ADR-TECH-STRAT-007 Rule 4.
- Every payload is validated against
  `process/BNK.RLVR.CAP.SUP.002.BEN/schemas/BNK.RLVR.RVT.SUP.002.BENEFICIARY_ANCHOR_UPDATED.schema.json`
  **before** publishing — fail-fast on contract drift.
- Cadence is configurable via env (`RELIEVER_CADENCE_MIN_PER_MINUTE`,
  `RELIEVER_CADENCE_MAX_PER_MINUTE`); default 1–10 events / minute.

### Query half — FastAPI HTTP endpoints
- `GET /anchors/{internal_id}` — canned `BeneficiaryAnchor` with
  `ETag` + `Cache-Control: max-age=60`; `404 ANCHOR_NOT_FOUND` for unknown
  ids; `304 Not Modified` on `If-None-Match` hit.
- `GET /anchors/{internal_id}/history?since_revision=N` — canned, PII-free
  `AnchorHistory` rows; `ETag` + `Cache-Control: max-age=0`; `304` honoured;
  `404` for unknown ids.
- `GET /health` — liveness probe.

### Fixtures (≥3, stable UUIDv7 ids)
| ID | Status | Notes |
|---|---|---|
| `018f8e10-0000-7000-8000-000000000001` | `ACTIVE` | full PII + 1 history row |
| `018f8e10-0000-7000-8000-000000000002` | `ARCHIVED` | full PII + 3 history rows |
| `018f8e10-0000-7000-8000-000000000003` | `PSEUDONYMISED` | PII null + 5 history rows incl. PSEUDONYMISED |

## Run it

### Prerequisites
- Python 3.12+
- `uv` (recommended) or `pip`
- Docker (only required for the publisher half — RabbitMQ)

### Local — query half only (no broker required)

```bash
cd sources/BNK.RLVR.CAP.SUP.002.BEN/stub
uv sync
uv run uvicorn reliever_beneficiary_identity_anchor_stub.app:app --port 54679
```

Then in another shell:

```bash
curl -i http://localhost:54679/health
curl -i http://localhost:54679/anchors/018f8e10-0000-7000-8000-000000000001
curl -i 'http://localhost:54679/anchors/018f8e10-0000-7000-8000-000000000003/history?since_revision=3'
```

### Local — both halves (publisher + query)

```bash
cd sources/BNK.RLVR.CAP.SUP.002.BEN/stub
docker compose up -d rabbitmq                  # waits ~10s for healthy state
RELIEVER_STUB_ACTIVE=true \
  uv run uvicorn reliever_beneficiary_identity_anchor_stub.app:app --port 54679
```

Subscribe with a smoke consumer:

```bash
docker run --rm --network host pivotalrabbitmq/perf-test \
  --uri amqp://admin:password@localhost:54879/ \
  --exchange sup.002.ben-events --exchange-type topic \
  --routing-key 'BNK.RLVR.EVT.SUP.002.BENEFICIARY_ANCHOR_UPDATED.#' \
  --consumers 1 --producers 0 --autodelete true --queue-pattern stub-watcher-%d
```

Or with `rabbitmqadmin`:

```bash
rabbitmqadmin --host localhost --port 54880 --username admin --password password \
  declare queue name=stub-watcher durable=false
rabbitmqadmin --host localhost --port 54880 --username admin --password password \
  declare binding source=sup.002.ben-events destination=stub-watcher \
    routing_key='BNK.RLVR.EVT.SUP.002.BENEFICIARY_ANCHOR_UPDATED.#'
rabbitmqadmin --host localhost --port 54880 --username admin --password password \
  get queue=stub-watcher count=5
```

| Service | Local URL |
|---|---|
| HTTP API | http://localhost:54679 |
| Swagger UI | http://localhost:54679/docs |
| RabbitMQ Management | http://localhost:54880 (admin / password) |
| RabbitMQ AMQP | amqp://admin:password@localhost:54879/ |

### Configuration

| Env var | Default | Effect |
|---|---|---|
| `RELIEVER_STUB_ACTIVE` | `false` | Master toggle — `true` enables publisher + serves HTTP. `false` halts both publication **and** HTTP serving (per DoD). |
| `RELIEVER_STUB_HTTP_ACTIVE` | `true` | Independent override for HTTP only. Set to `false` to silence the query half while keeping the publisher running. |
| `RELIEVER_AMQP_URL` | `amqp://admin:password@localhost:54879/` | RabbitMQ URL. |
| `RELIEVER_CADENCE_MIN_PER_MINUTE` | `1` | Lower bound of publish rate. |
| `RELIEVER_CADENCE_MAX_PER_MINUTE` | `10` | Upper bound. |
| `RELIEVER_HTTP_PORT` | `54679` | Uvicorn port. |

> **DoD note** — when `RELIEVER_STUB_ACTIVE=false`, the publisher loop is not
> started AND the HTTP endpoints short-circuit to `503` (lifespan still loads
> fixtures so `/health` keeps reporting `200`). Set both
> `RELIEVER_STUB_ACTIVE=true` and `RELIEVER_STUB_HTTP_ACTIVE=true` to enable
> everything.

## Tests

```bash
uv sync --extra dev
uv run pytest -q
```

The suite is **broker-free** — every test that exercises the publisher does
so via in-memory `aio-pika` mocks. Tests cover:

- Every transition kind produces a schema-valid payload (including
  `PSEUDONYMISED`).
- UUIDv7 envelope trio is present on every payload.
- Each canned `BeneficiaryAnchor` fixture covers the three `anchor_status`
  values (ACTIVE / ARCHIVED / PSEUDONYMISED).
- `GET /anchors/{id}` returns 200/304/404 per the api.yaml contract.
- `GET /anchors/{id}/history` honours `since_revision` and ETag/304.
- `STUB_ACTIVE=false` halts both halves (HTTP returns 503 from anchor routes,
  publisher loop is never started).

## Decommissioning

This stub is **temporary**. It is superseded by the real microservice scaffolded
under TASK-002 (foundation: MINT + GET) and the lifecycle tasks TASK-003 …
TASK-006. The decommissioning sequence:

1. The real `sources/BNK.RLVR.CAP.SUP.002.BEN/backend/` (Mode A microservice) ships.
2. The bus topology (exchange name, routing key, payload schema) is bit-identical
   to this stub (both read the same `process/BNK.RLVR.CAP.SUP.002.BEN/` contract).
3. Downstream consumers switch their broker target from the stub-hosted
   RabbitMQ to the real microservice's RabbitMQ once Mode A is integrated
   into the dev environment.
4. This folder is deleted by the PR that ships TASK-006 (anchor history
   projection — last lifecycle task). The deletion is bundled with the merge
   to keep `sources/` clean.

Until then, downstream-capability developers (SCO, ENR, ENV, DSH, VIE,
B2B.FLW, AUD, RET) **must** target this stub as their single source of
truth for `BNK.RLVR.CAP.SUP.002.BEN` contracts.
